Introduction

Buffer Overflow Overview

Buffer Overflow - caused by injecting information too large for the input to handle such that the code manipulates the CPU’s processing

They can

  • cause the program to crash
  • corrupt data
  • change the data structures in the programs runtime

Changing the data structures can change the process’s return address in memory which allows the attacker to execute commands with the privileges of the process.

Buffer Overflows are typically in Von-Neumann architecture.

Languages like C or C++ typically are very capable of producing a Buffer Overflow but languages like Java (which has a garbage collection) helps prevent BOF’s.

CPU Architecture

Von-Neumann architecture

  • Memory
  • Control Unit (CU)
  • Arithmetic Logic Unit (ALU)
  • Input/Output Unit (IO Unit)

ALU + CU = CPU (Central Processing Unit) - instructions and control flow

Instructions and data are fetched by the CU through the bus system

Bus System - connection between CPU, memory and IO unit

Memory

Primary Memory (RAM)

RAM - Defines a type of memory at a point in the memory at the memory address Cache - serves as a buffer to the CPU assuring instructions and date always get to the CPU

Before data and instructions get to the CPU, the RAM acts as the storage. The size of the RAM is the amount of data that can be stored in the processor.

Secondary Memory

SSD, Flash Drives, CD, etc.

Not directly accessed by the CPU, but via IO interfaces. Used to store memory that does not need to be processed at the moment.

Control Unit

Connected via bus system

Tasks

  • Reading data from the RAM
  • Saving data in RAM
  • Provide, decode and execute an instruction
  • Processing the inputs from peripheral devices
  • Processing of outputs to peripheral devices
  • Interrupt control
  • Monitoring of the entire system

Instruction Register (IR) - Contains all the instructions that the processor decodes and executes through the execution unit. Then the data is transfered to the ALU for calculation.

Registers - Store the data used during execution

CPU

Processes information and operations.

The CPU reads from memory and initiates data processing.

Different CPU architectures

  • x86/i386 - (AMD & Intel)
  • x86-64/amd64 - (Microsoft & Sun)
  • ARM - (Acorn)

Instruction Set Architecture (ISA) - CPU architectures, gives us the possibility to understand the unified behavior of machine code in assembly language concerning registers, data types, etc.

There are four different types of ISA:

  • CISC - Complex Instruction Set Computing
  • RISC - Reduced Instruction Set Computing
  • VLIW - Very Long Instruction Word
  • EPIC - Explicitly Parallel Instruction Computing

RISC

The smaller the instruction set, the faster the clock frequencies which enables faster execution.

CISC

Larger instruction set caused by recurring sequences of instructions combined into single instructions.

Addressing takes only an 8-bit number
